## Adversarial Attacks

- [CleverHans](https://github.com/cleverhans-lab/cleverhans) – Benchmark library for adversarial attacks and defenses.
- [Foolbox](https://github.com/bethgelab/foolbox) – Python toolbox for creating adversarial examples.
- [IBM Adversarial Robustness Toolbox (ART)](https://github.com/Trusted-AI/adversarial-robustness-toolbox) – Comprehensive library for attacks and defenses across models.
- [AdvTorch](https://github.com/BorealisAI/advertorch) – Toolbox for adversarial robustness in PyTorch.
- [TextAttack](https://github.com/QData/TextAttack) – Framework for NLP adversarial attacks.
- [DeepSec](https://github.com/deepsec/deepsec) – Benchmarking tool for adversarial attacks on neural networks.

## Defenses & Robustness

- [RobustBench](https://github.com/RobustBench/robustbench) – Leaderboard of adversarially robust models.
- [Adversarial Training (Madry Lab)](https://github.com/MadryLab/cifar10_challenge) – Classic examples of adversarial training.
- [ME-Net](https://github.com/UtkarshOJha/ME-Net) – Defense model against adversarial perturbations.
- [Randomized Smoothing](https://github.com/locuslab/smoothing) – Certified robustness technique for classification models.
- [Feature Squeezing](https://github.com/dhkim2810/feature-squeezing) – Traditional defense against adversarial examples.
  
## Model Security

- [ModelGuard](https://github.com/modelguard/modelguard) – Tools for securing ML supply chains and models.
- [AI Model Watermarking](https://github.com/topics/model-watermarking) – Tools and research on watermarking for model IP protection.
- [Model Extraction Tooling](https://github.com/jayatheeswaran0/model-extraction) – Scripts and examples for simulating model extraction attacks.
- [ONNX Runtime Security Guidelines](https://onnxruntime.ai/) – Secure inference recommendations for ONNX-based deployments.

## Data Security & Poisoning

- [CleanLab](https://github.com/cleanlab/cleanlab) – Identify noisy labels and corrupted datasets.
- [pytorch-poisoning](https://github.com/ManipalRoboticsClub/pytorch-poisoning) – Tools for dataset poisoning attacks.
- [TensorFlow Privacy](https://github.com/tensorflow/privacy) – Differential privacy training tools.
- [PySyft](https://github.com/OpenMined/PySyft) – Privacy-preserving data science with encrypted computation.
- [badNets Examples](https://github.com/Kooscii/badNets) – Classic dataset poisoning case studies.

## Red Teaming & Testing

- [OpenAI Red Teaming Network](https://openai.com/red-teaming-network) – Global collaboration for model-level security testing.
- [Microsoft Counterfit](https://github.com/Azure/counterfit) – Automation tool for adversarial ML security testing.
- [Anthropic Red Teaming Resources](https://www.anthropic.com/) – Tools and frameworks for systematic evaluations.
- [LLM Jailbreak Datasets](https://github.com/topics/jailbreak-prompts) – Public sets of adversarial prompts for testing.
- [PromptBench](https://github.com/microsoft/promptbench) – Benchmark for prompt injection and jailbreak robustness.

## Privacy-Preserving AI

- [OpenDP SmartNoise](https://github.com/opendp/smartnoise-sdk) – Differential privacy tools and synthetic data generators.
- [Federated Learning (Flower)](https://github.com/adap/flower) – Framework for federated and privacy-preserving learning.
- [FATE](https://github.com/FederatedAI/FATE) – Industrial-grade federated learning platform.
- [PyGrid / PySyft](https://github.com/OpenMined) – Secure, distributed data science tools.
- [Homomorphic Encryption Libraries](https://github.com/topics/homomorphic-encryption) – Tools for encrypted inference.

## Evaluation & Benchmarks

- [HELM](https://crfm.stanford.edu/helm/latest/) – Holistic evaluation of model safety and robustness.
- [RobustBench](https://github.com/RobustBench/robustbench) – Benchmark suite for robust classification models.
- [HarmBench](https://github.com/centerforaisafety/HarmBench) – Safety and harm classification benchmark for AI systems.
- [SafetyBench](https://github.com/centerforaisafety/SafetyBench) – Suite of tests for model safety.
- [OpenAI Evals](https://github.com/openai/evals) – Evaluation framework adaptable for adversarial and security testing.
- [MLSec Resources](https://github.com/topics/ml-security) – Community-driven lists of security-focused ML tools.

## Learning Resources

- [MIT ML Security Course](https://people.csail.mit.edu/madry/mlsec/) – Seminal course on adversarial ML.
- [AI Security Papers (CAIS)](https://www.safe.ai/papers) – Curated papers on robustness and adversarial threats.
- [Microsoft ML Security Research](https://www.microsoft.com/en-us/security/blog/) – Applied research on securing ML systems.
- [OpenAI Security Research](https://openai.com/research) – Findings on model vulnerabilities and defenses.
- [NIST AI Security Framework](https://www.nist.gov/) – Guidelines for secure and trustworthy AI.
